## 서울시청 수방 시스템 관리자 페이지

### 프로젝트 내용

지도를 기반하여 서울 시청 수방 정보를 제공해주는 웹 시스템

### 프로젝트 기간

#### 2023.08 ~ 2024.03

### 사용 기술

- Front : React, TypeScript, Zustand, React-Query, TailwindCSS
- 배포 : Jeus

### 구성 인원
- Front : 2명
- Designer : 1명

### 맡은 업무
- 서울 시청 수방 시스템 관리자 페이지 고도화 개발
  
### 성과
- 사용자에게 동적인 콘텐츠를 제공해 주기 위해 기존 legacy 코드로 구성된 JSP 페이지를 React로 Migration 하였습니다.
리팩토링 하는 과정에서 디자이너와 협업하여 재사용성 높은 UI 컴포넌트 설계와 렌더링 최소화 및 비동기 처리 프로세스를 개선하여 사용자 경험과 웹 성능을 향상시켰습니다.
- 기존 협업 프로세스를 전환 및 개선하였습니다. 이를 통해 효율적인 브랜치 관리와 컨벤션에 맞는 커밋 메시지를 작성할 수 있게 되어 개발 효율성 향상 및 코드 품질을 개선할 수 있었습니다.

### 세부내용
**JSP 페이지 React로 Migration**
- boilerplate code로 구성된 기존 JSP 페이지를 리팩토링하는 과정에서 중복된 UI 요소를 최소화하고 재사용할 수 있는 컴포넌트를 구현하기 위해 React로 Migration 작업을 추진하여 유지보수성을 높이고 사용자에게 동적인 UI를 제공하였습니다.
UI 컴포넌트 설계 및 개발
- Atomic Design Pattern을 기반으로 디자이너와 커뮤니케이션을 통해 컴포넌트 분류 기준점을 찾아 단계별로 컴포넌트를 설계하여 컴포넌트 재사용성과 유지보수성 증가하였습니다.

**상태 관리 최적화**
- React-Query를 활용하여 데이터 패칭 작업을 효율적으로 개선하였습니다.
- React-Query의 캐싱 기능을 사용하여 중복 요청을 방지하고 웹 성능을 최적화하였습니다.
- Zustand로 전역 상태 관리를 하여 props 복잡도 개선 및 불필요한 렌더링을 최소화하였습니다.

**비동기 통신 개선**
- Axios를 도입하여 REST API 통신을 간결하게 처리하고, Interceptor를 활용하여 한 곳에서 에러 핸들링을 하도록 개선하였습니다.

**사용자 경험 개선**
- React.lazy와 Suspense를 활용하여 Code Splitting 하여 기존 페이지 로딩 속도(FCP) 2.5초에서 1.3초로 줄였습니다.

**협업 프로세스 전환 및 개선**
- 효율적인 branch 관리를 위해 기존에 사용했던 SVN을 Git으로 전환하였습니다.
- Github Flow를 도입하여 협업 프로세스와 코드 관리를 개선하여 팀원 간의 협업을 원활하게 진행하게 되었습니다.
- Commit Message Convention을 도입하여 개발자들 간의 코드 변경 이력을 일관되게 관리하였습니다.

**비동기 통신 개선**

- 기존 복잡한 ajax 대신 Axios를 도입하여 REST API 통신을 간결하게 처리하고, Interceptor를 활용하여 에러 핸들링을 개선하였습니다.
- Axios 적용으로 코드 가독성을 향상하고 유지보수성을 높였습니다.
